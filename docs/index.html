UniOps Documentation

Agent-First 统一服务器运维平台。通过 AI Agent 控制集群节点，实现 nginx 反向代理 + 后端服务的自动化部署与管理。

Go 1.26 React 19 SQLite embedded SSH proxy License MIT

Overview

UniOps 是一个面向中小团队的堡垒机 + 运维平台，核心理念是 Agent-First：所有运维操作都可以通过 AI Agent 对话完成，Agent 通过 SSH 执行命令、部署服务、管理配置。

SSH Proxy

Web Terminal 直连目标服务器，支持密码/密钥认证，自动密钥部署。

AI Agent

支持 Claude / OpenAI / Custom(Ollama) 三种 Provider，对话式驱动部署。

Auto Deploy

一键部署后端二进制 + Nginx 反向代理 + Systemd 服务管理。

Session Audit

所有 Terminal 操作自动录制回放，操作日志按天分文件留痕。

File Manager

通过 SFTP 浏览、上传、下载远程服务器文件。

Monitoring

实时 CPU / 内存 / 磁盘 / 网络监控，5 秒自动刷新。

Quick Start

Docker (推荐)

docker-compose up -d

访问 http://localhost:6020，默认账号 root / root。

从源码构建

# 前端
cd web && npm ci && npm run build && cd ..

# 后端
CGO_ENABLED=0 go build -ldflags "-s -w -X main.version=dev" -o uniops ./cmd/uniops

# 运行
./uniops

集群测试环境

启动 3 节点 SSH 集群模拟生产环境：

docker compose -f docker-compose.cluster.yml up -d --build

# 节点: root / uniops123
# node-1: localhost:2221
# node-2: localhost:2222
# node-3: localhost:2223

Server Management

服务器是整个平台的核心资源。每个 Server 记录连接信息和认证凭据，支持密码和 SSH 密钥两种认证方式。

添加服务器

在 Servers 页面填写主机信息。如果使用密钥认证，需要先在 SSH Keys 中生成密钥对并部署到目标机器。

自动密钥部署

平台内置 RSA 密钥对生成器，一键将公钥部署到目标服务器的 ~/.ssh/authorized_keys：

POST /api/sshkeys/generate     → 生成密钥对
POST /api/sshkeys/deploy       → 通过密码部署公钥到目标

Web Terminal

基于 xterm.js 的全功能 Web 终端，通过 WebSocket 桥接 SSH 会话。支持：

  • PTY 终端模拟（xterm-256color）
  • 终端窗口自适应缩放
  • 全操作自动录制回放（Session Audit）

File Manager

通过 SFTP 协议浏览远程文件系统。支持目录导航、文件上传/下载/删除、创建目录。路径面包屑导航支持快速跳转。

Deployment

部署系统通过 SSH 在目标服务器上执行一系列命令，完成自动化部署。支持三种部署类型：

Type说明操作
backend后端服务下载二进制 → 创建 systemd unit → 启动服务
nginx反向代理安装 nginx → 写入 upstream 配置 → 启用站点
full全量部署backend + nginx 顺序执行

部署流程

1. SSH 连接目标服务器
2. mkdir -p /opt/{service_name}
3. curl 下载二进制到 /opt/{service_name}/
4. 创建 /etc/systemd/system/{service_name}.service
5. systemctl daemon-reload && enable && restart
6. 健康检查等待 (最多 30s)
7. (nginx) 写入 upstream + server block → nginx -t → restart

WebSocket 实时日志

部署过程中通过 WebSocket 推送实时日志，前端展示在 Deploy 页面的日志面板中：

ws://host:6020/api/deploy/{id}/ws

AI Agent

AI Agent 是 UniOps 的核心差异化功能。Agent 通过 Tool Call 循环自动执行 SSH 命令和部署操作。

Claudeapi.anthropic.comclaude-3-5-sonnet

Agent Tool Calls

Agent 通过 ```tool 代码块输出结构化指令，后端解析并执行后将结果回传 Agent，最多循环 5 轮：

```tool
{"action":"exec","params":{"command":"systemctl status nginx"}}
```

```tool
{"action":"deploy_full","params":{
  "service_name":"myapp",
  "binary_url":"https://github.com/.../releases/download/v1.0/myapp-linux-amd64",
  "app_port":8080,
  "domain":"myapp.example.com",
  "nginx_port":80
}}
```

Nginx Reverse Proxy

部署时自动生成的 Nginx 配置包含：

upstream {service_name}_backend {
    server 127.0.0.1:{app_port};
    keepalive 32;
}

server {
    listen {nginx_port};
    server_name {domain};

    # WebSocket 支持
    location /ws/ {
        proxy_pass http://{service_name}_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400s;
    }

    # API + 静态资源
    location / {
        proxy_pass http://{service_name}_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

Authentication

基于 JWT 的认证系统。Access Token 15 分钟过期，Refresh Token 7 天过期。

POST /api/auth/login
Body: {"username": "root", "password": "root"}
Response: {"access_token": "eyJ...", "user": {...}}

所有受保护 API 需要 Authorization: Bearer {token} 请求头。

Session Audit

所有 SSH Terminal 操作自动录制。每条 Session 记录包含操作者、目标服务器、起止时间和完整回放数据（Base64 编码的 JSON）。

在前端 Sessions 页面可以回放任意历史操作，按时间线逐步展示输入/输出。

Operation Logs

所有 API 操作自动记录到日志文件，按天分文件存储在 logs/ 目录：

logs/
├── 2026-07-01.jsonl
├── 2026-07-02.jsonl
└── ...

每条日志为 JSON Lines 格式：

{"timestamp":"...","user":"root","user_id":1,"action":"POST /api/deploy","method":"POST","path":"/api/deploy","status":201,"ip":"172.28.0.1:54321"}

日志管理

操作说明
按天查看选择日期标签，查看当天全部日志
搜索按 user / action / keyword 过滤
范围清理选择起止日期，批量删除日志文件
